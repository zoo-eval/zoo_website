From the Wild Web to the ZOO

Benchmarking web agents with a realistic simulator — a reproducible, interconnected environment of 13+ web applications for evaluating capability and security of AI agents.

Brian Grinstead (Mozilla) · Mariana Meireles · Christoph Kerschbaumer (Mozilla) · Cameron Allen (UC Berkeley)
GitHub → the_zoo GitHub → zoo-eval

What's New

01Realistic simulated web — 13+ interconnected apps (email, git, e-commerce, kanban, wiki…) in one isolated Docker network under the .zoo domain.
02Full backend observability — Direct access to databases, mail servers, and service state for ground-truth verification of agent behavior.
03Deterministic reproducibility — Snapshot-based resets ensure every experiment starts from identical conditions, unlike the live web.
04Cross-app security testing — Shared email, OAuth/OIDC, and databases enable multi-step attack chains (e.g., email-triggered prompt injection → data exfiltration).
05Flexible scenario constructor — Universes, Tasks, and Scenes API for building custom benchmarks; harness-agnostic with multi-agent and LLM-as-judge support.

Architecture

ZOO runs inside an air-gapped Docker network. A forward proxy (Squid) mediates all agent traffic, Caddy handles HTTPS, and CoreDNS resolves .zoo domains. Shared backend services — Stalwart mail, Ory Hydra (OIDC), MySQL, PostgreSQL, Redis — let unmodified open-source apps communicate just as they would on the real web.

Scenario Constructor

ZOO's evaluation framework is built on three concepts: Universes define the world (which apps and personas exist), Tasks define what agents must achieve with weighted sub-tasks and evaluation criteria, and Scenes bring dynamism through timed events, HTTP triggers, and content injection — enabling reproducible attack scenarios. The system is harness-agnostic, supports heterogeneous multi-agent setups, and provides three evaluation modes: string matching, custom Python functions, and LLM-as-judge.
